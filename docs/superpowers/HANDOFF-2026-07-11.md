## What we're building (one paragraph)

**Clara**, a voice AI care line. Caller dials a real phone number → Twilio (Nolan's
personal number) → Vapi (handles STT/TTS/turn-taking) → **our TypeScript server** exposing
an OpenAI-compatible `/chat/completions` SSE endpoint that Vapi uses as its custom LLM.
Our server carries all the intelligence: guardrail-first triage, a provider-swappable model
gateway, per-caller memory, and grounded tool calls to the upstream health data API.

## Components to build (~7 small TS files — see spec §4)

- `src/server.ts` — Fastify, `/chat/completions` SSE (Vapi custom-LLM contract)
- `src/guardrail.ts` — triage pre-check + scripted 911/988 responses + keyword fail-safe
- `src/gateway.ts` — `complete(messages, tools)`, Anthropic default, OpenAI swap via `MODEL_PROVIDER`
- `src/careApi.ts` — typed care-API client; auth header, base URL, 429 `retry_after`, **mock mode**
- `src/tools.ts` — tool schemas + dispatch: `find_clinics`, `med_price`, `care_info`, `housing_check`
- `src/memory.ts` — JSON-file caller profiles keyed by E.164 phone
- `src/prompts.ts` — Clara persona system prompt + personalization injection
- `scripts/smoke.ts` — posts fake Vapi payloads (normal turn + emergency turn); asserts the
  emergency path NEVER reaches the LLM

Hackathon simplifications are intentional: JSON file, not a DB; read-only care-API
integration (no event subs / webhooks); single assistant.

## The guardrail pipeline (the safety centerpiece — get this right)

Every caller utterance, in order: (1) load caller profile by phone; (2) **guardrail** — call
care-API triage (`GET /med-price?symptoms=<utterance>`, read the deterministic `triage`
object); if `hardEscalate` is `911` or `988`, stream back a FIXED scripted escalation — the
LLM is never consulted on that path; (3) otherwise LLM turn with triage band + profile +
tools; (4) save salient facts. **Fail-safe, never fail-open:** if the care API is
unreachable, a local keyword screen (chest pain, trouble breathing, suicide, overdose,
stroke signs) triggers the same script. Degraded ≠ unguarded.

## BLOCKER: care-API token

- The care API needs a scoped bearer token (`care:read`, plus `calls:read` for call events).
- Nolan is **logged into the care-API operator portal** (has Ops/Keys/System tabs).
  The magic-link login for `ss-agent@agentmail.to` is **waitlisted** — do NOT keep retrying
  it; it just returns "on the waitlist" until an operator approves.
- **Unblock path:** Nolan mints a token in the portal's **Keys** tab (scopes `care:read` +
  `calls:read`, label `clara-hackathon`) and pastes the `xapp-...` token → put it in
  `.env` as `CARE_API_TOKEN`. Shown once.
- **Until the token exists, build and test against `CARE_API_MOCK=1`** — `careApi.ts` returns
  realistic fixtures so the whole pipeline (guardrail, tools, memory, demo) works end-to-end.
  The real token then hot-swaps in with zero code change.
